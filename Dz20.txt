1. Описание моделей и связей между ними
Модель Author (Автор)
class Author(models.Model):
    name = models.CharField(max_length=100)      # Имя автора
    bio = models.TextField()                     # Биография


Назначение: Хранит информацию об авторах книг.
Связи:
	•	author → Book: один автор может написать много книг (One-to-Many через ForeignKey в Book)

Модель Genre (Жанр)
class Genre(models.Model):
    name = models.CharField(max_length=100)       # Название жанра
    description = models.TextField()              # Описание жанра


Назначение: Классифицирует книги по жанрам.
Связи:
	•	Genre ⇄ Book: многие ко многим (ManyToMany). Одна книга может относиться к нескольким жанрам, и один жанр может объединять много книг.

Модель Publisher (Издательство)
class Publisher(models.Model):
    name = models.CharField(max_length=100)       # Название издательства
    location = models.TextField()                 # Город расположения


Назначение: Хранит информацию об издательствах.
Связи:
	•	publisher → Book: одно издательство может выпускать много книг (One-to-Many через ForeignKey в Book)

Модель Store (Магазин)
class Store(models.Model):
    name = models.CharField(max_length=100)       # Название магазина
    city = models.CharField(max_length=100)       # Город


Назначение: Хранит информацию о книжных магазинах.
Связи:
	•	Store ⇄ Book: многие ко многим (ManyToMany). Книга может продаваться в нескольких магазинах, и в магазине может продаваться много книг.

Модель Review (Отзыв)
class Review(models.Model):
    rating = rating = models.PositiveIntegerField(default=7) 	# Оценка (1-10)
    published_date = models.DateField()           		# Дата публикации отзыва
    review = models.TextField()                   		# Текст отзыва
    book = models.ForeignKey(Book, on_delete=models.SET_NULL,	# Может быть пустым при удалении
        null=True,
        blank=True,
        related_name='reviews'
    )



Назначение: Хранит отзывы читателей на книги.
Связи:
	•	book → Review: одна книга может иметь много отзывов (One-to-Many через ForeignKey в Review). При удалении книги все её отзывы удаляются (CASCADE).

Модель Book (Книга)
class Book(models.Model):
    title = models.CharField(max_length=200)                     	# Название
    author = models.ForeignKey(Author, on_delete=models.CASCADE, related_name='books') # при удалении в таблице автора удаляется его книга
    published_date = models.DateField()                          	# Дата публикации
    description = models.TextField()                             	# Описание
    genres = models.ManyToManyField(Genre, related_name='books') 	# Жанры
    stores = models.ManyToManyField(Store, related_name='books') 	# Магазины
    publisher = models.ForeignKey(Publisher, on_delete=models.SET_NULL,	# Поле может быть пустым при удалении ссылочной таблицы
      null=True,
      blank=True,
      related_name='books'
    )

Назначение: Центральная модель, объединяющая все связи.
Связи:


Связь			Тип		Поле		  Объяснение
Книга → Автор		Many-to-One	author		  У книги один автор
Книга → Издательство	Many-to-One	publisher	  У книги одно издательство
Книга → Жанры		Many-to-Many	genres		  Книга может иметь несколько жанров
Книга → Магазины	Many-to-Many	stores		  Книга может продаваться в нескольких магазинах
Книга → Отзывы		One-to-Many	через Review.book У книги может быть много отзывов

2. Описание выполненных запросов
Запрос 1: Книги издательств из определённого города
books_nsk = Book.objects.select_related('publisher').filter(publisher__location='Новосибирск')


Что делает: Находит все книги, выпущенные издательствами, расположенными в Новосибирске.
Логика:
	•	publisher__location — обращение к полю location связанной модели Publisher
	•	select_related() — оптимизация: одним запросом подгружает данные издательства
Результат: Список книг с информацией об издательстве.

Запрос 2: Книги, продающиеся в магазинах Москвы
books_msk = Book.objects.filter(stores__city='Москва').distinct().prefetch_related('stores')


Что делает: Находит все книги, которые продаются хотя бы в одном магазине Москвы.
Логика:
	•	stores__city — фильтрация по городу магазина через связь ManyToMany
	•	.distinct() — убирает дубликаты (книга может продаваться в нескольких московских магазинах)
	•	prefetch_related() — оптимизация: подгружает все магазины для книг одним дополнительным запросом
Результат: Список книг с информацией о магазинах, где они продаются.

Запрос 3: Книги со средней оценкой выше 7.5
high_rated = Book.objects.annotate(
    avg=Avg('reviews__rating')
).filter(avg__gt=7.5)


Что делает: Находит книги, средняя оценка которых превышает 7.5.
Логика:
	•	.annotate(avg=Avg('reviews__rating')) — добавляет к каждой книге поле avg со средней оценкой
	•	reviews__rating — обращение к полю rating связанных отзывов
	•	.filter(avg__gt=7.5) — оставляет только книги с оценкой выше 7.5
Результат: Книги с подсчитанной средней оценкой.

Запрос 4: Количество книг в каждом магазине
stores_cnt = Store.objects.annotate(cnt=Count('books')).order_by('-cnt')


Что делает: Подсчитывает количество книг, продающихся в каждом магазине, и сортирует по убыванию.
Логика:
	•	.annotate(cnt=Count('books')) — добавляет поле cnt с количеством книг
	•	Count('books') — подсчитывает связанные объекты через ManyToMany
	•	.order_by('-cnt') — сортировка от большего к меньшему
Результат: Список магазинов с указанием количества книг.

Запрос 5: Магазины с книгами после 1920 года
stores_1920 = Store.objects.filter(
    books__published_date__gt=date(1920, 1, 1)
).annotate(cnt=Count('books')).distinct().order_by('-cnt')


Что делает: Находит магазины, где продаются книги, изданные после 1920 года, сортирует по количеству таких книг.
Логика:
	•	.filter(books__published_date__gt=date(1920, 1, 1)) — оставляет только магазины с книгами, изданными после 1920
	•	.annotate(cnt=Count('books')) — подсчитывает количество таких книг
	•	.distinct() — убирает дубликаты магазинов
	•	.order_by('-cnt') — сортировка по убыванию количества
Результат: Список магазинов, отсортированных по количеству "новых" книг.

3. Оптимизация запросов
Проблема N+1 запросов
Без оптимизации при выборке книг и последующем обращении к связанным данным Django делает отдельный запрос для каждой записи.
Пример проблемы:
# Без оптимизации
books = Book.objects.all()
for book in books:
    print(book.publisher.name)  # Делает N дополнительных запросов!
select_related() — для ForeignKey и OneToOne
books_nsk = Book.objects.select_related('publisher').filter(publisher__location='Новосибирск')

Что делает: Выполняет один SQL-запрос с LEFT OUTER JOIN для подгрузки данных издательства.
Когда использовать: Для связей ForeignKey и OneToOneField.
prefetch_related() — для ManyToMany и обратных связей
books_msk = Book.objects.filter(stores__city='Москва').distinct().prefetch_related('stores')

Что делает: Выполняет два SQL-запроса:
	1	Выборка книг
	2	Выборка всех связанных магазинов для этих книг
Затем Django связывает данные в памяти.
Когда использовать: Для связей ManyToMany и обратных связей ForeignKey (например, book.reviews.all()).

books = Book.objects.select_related('publisher').prefetch_related('reviews', 'stores')

 📖 Пошаговые инструкции по созданию тестовых данных для издательств, магазинов и книг через административную панель.

Шаг 1: Запусти сервер
bash
python manage.py runserver


Шаг 2: Зайди в админ-панель
	1	Открой браузер и перейди по адресу: http://127.0.0.1:8000/admin
	2	Введи логин и пароль суперпользователя предоставленные разработчиком. После этого открывается страница Рис. AdminPanel.png


🏢 1. Создание издательств (Publisher)
Шаг 1: Перейди в раздел "Publishers"
	•	В левой панели найди раздел PUBLISHERS → Publishers
	•	Нажми кнопку + Add
Шаг 2: Заполни форму


Поле
Значение
Name
Название издательства (например, "Эксмо", "АСТ", "Питер")
Location
Город расположения (например, "Москва", "Санкт-Петербург", "Новосибирск")
Шаг 3: Сохрани
	•	Нажми SAVE (внизу страницы)

🏪 2. Создание магазинов (Store)
Шаг 1: Перейди в раздел "Stores"
	•	В левой панели найди раздел STORES → Stores
	•	Нажми + Add
Шаг 2: Заполни форму


Поле
Значение
Name
Название магазина (например, "Дом книги на Арбате", "Читай-город")
City
Город расположения (например, "Москва", "Санкт-Петербург")
Шаг 3: Сохрани
	•	Нажми SAVE

🏷️ 3. Создание жанров (Genre)
Шаг 1: Перейди в раздел "Genres"
	•	В левой панели найди раздел GENRES → Genres
	•	Нажми + Add
Шаг 2: Заполни форму


Поле
Значение
Name
Название жанра (например, "Роман", "Детектив", "Фантастика", "Поэзия")
Description
Описание жанра (необязательно, для справки)
Шаг 3: Сохрани
	•	Нажми SAVE

👤 4. Создание авторов (Author)
Шаг 1: Перейди в раздел "Authors"
	•	В левой панели найди раздел BOOKS → Authors
	•	Нажми + Add
Шаг 2: Заполни форму


Поле
Значение
Name
Имя автора (например, "Лев Толстой", "Джордж Оруэлл")
Bio
Краткая биография (необязательно)
Шаг 3: Сохрани
	•	Нажми SAVE

📚 5. Создание книг (Book)
Шаг 1: Перейди в раздел "Books"
	•	В левой панели найди раздел BOOKS → Books
	•	Нажми + Add
Шаг 2: Заполни форму (основная информация)


Поле
Значение
Пример

Шаг 3: Выбери жанры (Genres)
	•	В поле Genres нажми и удерживай Cmd (на Mac) или Ctrl (на Windows)
	•	Выбери один или несколько жанров
Шаг 4: Выбери магазины (Stores)
	•	В поле Stores выбери магазины, где продаётся книга
	•	Можно выбрать несколько
Шаг 5: Сохрани
	•	Нажми SAVE

⭐ 6. Создание отзывов (Review)
Шаг 1: Перейди в раздел "Reviews"
	•	В левой панели найди раздел REVIEWS → Reviews
	•	Нажми + Add
Шаг 2: Заполни форму


Поле
Значение
Rating
Оценка от 1 до 10

Дата публикации отзыва (например, 2025-01-01)

Текст отзыва
Book
Выбери книгу из списка
Шаг 3: Сохрани
	•	Нажми SAVE

📋 Быстрый способ: создание нескольких отзывов
